windows服务器关闭23，135，335端口

发表于 2018-10-19 | 分类于 windows

23端口

端口说明：23端口主要用于Telnet(远程登录)服务，是Internet上普遍采用的登录和仿真程序。同样需要设置客户端和服务器端，开启Telnet服务的客户端就可以登录远程Telnet服务器，采用授权用户名和密码登录。登录之后，允许用户使用命令提示符窗口进行相应的操作。在Windows中可以在命令提示符窗口中，键入“Telnet”命令来使用Telnet远程登录。

操作建议：利用Telnet服务，黑客可以搜索远程登录Unix的服务，扫描操作系统的类型。而且在Windows 2000中Telnet服务存在多个严重的漏洞，比如提升权限、拒绝服务等，可以让远程服务器崩溃。Telnet服务的23端口也是TTS(Tiny Telnet Server)木马的缺省端口。所以，建议关闭23端口。

135端口

端口说明：135端口主要用于使用RPC(Remote Procedure Call，远程过程调用)协议并提供DCOM(分布式组件对象模型)服务，通过RPC可以保证在一台计算机上运行的程序可以顺利地执行远程计算机上的代码;使用DCOM可以通过网络直接进行通信，能够跨包括HTTP协议在内的多种网络传输。

端口漏洞：相信去年很多Windows 2000和Windows XP用户都中了“冲击波”病毒，该病毒就是利用RPC漏洞来攻击计算机的。RPC本身在处理通过TCP/IP的消息交换部分有一个漏洞，该漏洞是由于错误地处理格式不正确的消息造成的。该漏洞会影响到RPC与DCOM之间的一个接口，该接口侦听的端口就是135。

操作建议：为了避免“冲击波”病毒的攻击，建议关闭该端口。

445端口

端口说明445端口是一个毁誉参半的端口，有了它我们可以在局域网中轻松访问各种共享文件夹或共享打印机，但也正是因为有了它，黑客们才有了可乘之机，他们能通过该端口偷偷共享你的硬盘，甚至会在悄无声息中将你的硬盘格式化掉!

markdown语法

发表于 2018-10-17 | 分类于 技巧

vscode 支持一边编辑一边预览，有2种方法：

  1. CTRL + shift + P 调出命令框，输入>markdown 找到markdown：open 》preview的选项选中
  2. 先按ctrl + k ,然后放掉，接着按 V ，调出实时预览框

markdown 语法

1. 标题

行首加上#号+空格 即可定义标题，#号的个数代表标题的级数，一共6级。

2. 列表

  • 无序列表

    • 可用星号（*），加号（+），减号（—）作为标记，后面跟空格。3者可混用。

  • 有序列表

    • 数字加上句号（1. ），数字不影响显示效果

      1. 11
      2. 222
      3. 22222

  • 列表的缩进和混排

    • 若需缩进，在*，+，—前面加 空格

  • 记事列表

    • 使用 - [x] 开头的表示已完成的记事
    • 使用 - [ ] 开头的表示待办记事
      • 完成记事
      • 待办记事

3. 嵌套块的引用

> 使用“>”可以表示嵌套的引用

4. 文字效果和分割线

斜体
粗体
++下划线++
删除线
==文本高亮==

一行中用三个以上的星号 (*)、减号 (-) 或是下划线 (_) 来建立一个分隔线，需要独占一行，中间可以有空格，但行内不能包含其他字符

5. 链接

  • 文本连接
    格式：[文本] (链接地址 “可选说明文字，鼠标放上提示的文字”)

    baidu

  • 图片链接
    格式：！[文本] (链接地址 “可选说明文字，鼠标放上提示的文字”)

6. 文字高亮和代码块

  • 需要高亮的文本，使用 号包括，例如高亮`
  • 引用代码块使用两组
